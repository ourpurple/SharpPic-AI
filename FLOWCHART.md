## 启动流程

```
main.py
  │
  ├─► 1. 加载配置 (config.load)
  │     ├─► 读取 ~/.sharppic/config.json
  │     ├─► 加载内置加密配置 (builtin_config)
  │     └─► 合并默认配置
  │
  ├─► 2. 初始化 PyQt6 应用
  │     ├─► 设置应用名称和图标
  │     └─► 应用主题样式
  │
  ├─► 3. 创建主窗口 (MainWindow)
  │
  ├─► 4. 首次运行检查
  │     └─► 如果未配置 API Key → 打开设置对话框
  │
  └─► 5. 启动事件循环
```

## 图片加载流程

```
用户操作
  │
  ├─► 方式1: 点击"选择文件"按钮
  │     └─► QFileDialog 选择图片文件
  │
  ├─► 方式2: 拖拽图片到显示区域
  │     └─► ImageDropLabel 接收拖放事件
  │
  └─► 方式3: Ctrl+V 粘贴
        ├─► 检测剪贴板内容
        ├─► 如果是文件URL → 加载文件
        └─► 如果是图片数据 → 保存到临时文件
              │
              └─► _load_source(path)
                    ├─► 加载为 QPixmap
                    ├─► 显示在左侧面板
                    ├─► 清空右侧结果
                    └─► 更新状态栏
```

## 图片处理流程

```
用户点击"开始处理"
  │
  ├─► 1. 验证前置条件
  │     ├─► 检查是否已选择图片
  │     └─► 检查 API Key 配置
  │           └─► 未配置 → 打开设置对话框
  │
  ├─► 2. 创建工作线程 (ProcessWorker)
  │     ├─► 传入图片路径
  │     └─► 连接信号槽
  │           ├─► text_chunk → 实时文本输出
  │           ├─► finished → 处理完成
  │           └─► error → 错误处理
  │
  ├─► 3. 启动后台处理
  │     └─► ProcessWorker.run()
  │           │
  │           └─► process_image_stream()
  │                 │
  │                 ├─► 根据 api_provider 路由
  │                 │     │
  │                 │     ├─► "openai" → OpenAI 兼容流程
  │                 │     │     │
  │                 │     │     ├─► 构建消息 (_build_messages)
  │                 │     │     │     ├─► 系统提示词 (prompts.py)
  │                 │     │     │     ├─► 用户消息
  │                 │     │     │     └─► 图片 base64 编码
  │                 │     │     │
  │                 │     │     ├─► 调用 OpenAI API (流式)
  │                 │     │     │     └─► 逐块接收响应
  │                 │     │     │           ├─► 文本内容 → 实时显示
  │                 │     │     │           └─► 图片数据 → 收集
  │                 │     │     │
  │                 │     │     └─► 提取结果图片
  │                 │     │           ├─► 优先级1: model_extra 中的图片
  │                 │     │           ├─► 优先级2: 文本中的 base64
  │                 │     │           └─► 优先级3: 文本中的 URL
  │                 │     │
  │                 │     └─► "gmicloud" → gmicloud 流程
  │                 │           │
  │                 │           ├─► 图片预处理
  │                 │           │     └─► pad_to_aspect_ratio()
  │                 │           │           ├─► 检测或指定宽高比
  │                 │           │           └─► 添加白边适配
  │                 │           │
  │                 │           ├─► 提交请求 (_submit_request)
  │                 │           │     ├─► 构建 Gemini 格式 payload
  │                 │           │     └─► POST 到队列 API
  │                 │           │           └─► 返回 request_id
  │                 │           │
  │                 │           ├─► 轮询状态 (每2秒)
  │                 │           │     ├─► GET /requests/{id}
  │                 │           │     ├─► status = "success" → 继续
  │                 │           │     ├─► status = "failed" → 抛出错误
  │                 │           │     └─► 超时 (300秒) → 抛出错误
  │                 │           │
  │                 │           └─► 下载结果图片
  │                 │                 ├─► 从 media_urls 获取 URL
  │                 │                 └─► 下载并转 base64
  │                 │
  │                 └─► 返回 base64 图片数据
  │
  ├─► 4. 处理完成回调
  │     ├─► 解码 base64 为 QPixmap
  │     ├─► 显示在右侧面板
  │     ├─► 启用"保存图片"按钮
  │     └─► 更新状态栏
  │
  └─► 5. 错误处理
        ├─► 显示错误对话框
        └─► 更新状态栏
```

## 配置管理流程

```
配置系统 (config.py)
  │
  ├─► 配置文件位置: ~/.sharppic/config.json
  │
  ├─► 配置加载优先级 (从低到高):
  │     1. 硬编码默认值 (_DEFAULTS)
  │     2. 内置加密配置 (builtin_config.py)
  │     3. 用户配置文件 (config.json)
  │
  ├─► 主要配置项:
  │     ├─► api_provider: "openai" | "gmicloud"
  │     ├─► api_base_url: OpenAI 接口地址
  │     ├─► api_key: OpenAI API Key
  │     ├─► model_name: 模型名称
  │     ├─► gmi_api_key: gmicloud API Key
  │     ├─► gmi_image_size: "512"|"1K"|"2K"|"4K"
  │     ├─► gmi_aspect_ratio: 宽高比设置
  │     └─► save_directory: 默认保存目录
  │
  └─► 设置对话框 (SettingsDialog)
        ├─► 接口类型切换
        │     └─► 动态显示对应配置项
        ├─► 测试连接功能
        │     ├─► OpenAI: 发送测试消息
        │     └─► gmicloud: 验证 API Key
        ├─► 获取模型列表 (OpenAI)
        └─► 保存配置到文件
```

## 线程模型

```
主线程 (GUI)
  │
  ├─► UI 渲染和事件处理
  ├─► 用户交互响应
  └─► 信号槽连接
        │
        └─► 创建工作线程
              │
              ▼
        工作线程 (ProcessWorker)
              │
              ├─► API 请求处理
              ├─► 数据流式接收
              └─► 发送信号到主线程
                    │
                    ├─► text_chunk (实时文本)
                    ├─► finished (完成结果)
                    └─► error (错误信息)
```

## 关键技术点

1. **多接口支持**: 通过 api_provider 配置动态路由到不同的 API 实现
2. **流式处理**: OpenAI 接口支持流式响应，实时显示处理进度
3. **队列轮询**: gmicloud 接口使用异步队列模式，定期轮询状态
4. **图片适配**: gmicloud 需要特定宽高比，自动添加白边适配
5. **多种输入**: 支持文件选择、拖拽、剪贴板粘贴三种输入方式
6. **线程安全**: 使用 QThread 和信号槽机制保证 UI 响应性
7. **配置分层**: 支持默认配置、内置配置、用户配置三层覆盖
8. **错误恢复**: 完善的异常处理和用户提示机制
